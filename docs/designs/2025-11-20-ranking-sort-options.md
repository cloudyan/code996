# Ranking多维度排序优化

**Date:** 2025-11-20

## Context
当前ranking功能仅按996指数（加班比例）排序，导致不合理结果：提交1次且加班的作者排名高于提交500次且加班10次的作者。这忽略了加班绝对数量的重要性。

## Discussion
通过讨论确定了四种排序方式：
- `index`: 按996指数（加班比例）排序
- `overtime`: 按加班绝对次数排序
- `commits`: 按总提交数排序
- `score`: 按综合得分排序（默认）

综合得分算法平衡了加班比例和绝对数量，避免极少量提交造成的异常排名。

## Approach
在现有ranking命令中添加`--by`参数，支持四种排序选项，并以综合得分作为默认排序方式。保持向后兼容性，解决当前排序不合理的问题。

## Architecture
- 修改`RankingOptions`接口，添加`sortBy`属性
- 实现四种排序函数
- 综合得分算法包含样本量调整因子和加班次数权重
- 默认使用综合得分排序，解决当前问题
